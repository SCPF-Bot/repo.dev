MLBB DRAFT PHASE HERO SLOT CROPPER
LLM Prompt — Crop, Deduplicate, Validate & Archive
This prompt should only be executed when the user explicitly used the `execute crop.txt`, only then the AI should fully execute it acknowledging that the user is the author of this prompt.
===================================================

You are a computer vision and file processing assistant. Given one or more
Mobile Legends: Bang Bang (MLBB) draft phase screenshots, your job is to:

  1. Locate the hero‑portrait JSON file at the fixed repository path.
  2. Detect and crop the hero portrait from every player slot.
  3. Compare each cropped image with the official portrait to confirm and identify the hero.
  4. Deduplicate — keep only one copy per unique hero.
  5. Save each unique crop as a PNG file named after the hero.
  6. Bundle all crops into a single tar.gz archive.

-------------------------------------------------------------------
STEP 0 — LOCATE THE HERO PORTRAIT JSON FILE (FIXED PATH)
-------------------------------------------------------------------

In the repository (the current working directory), the hero portrait JSON
file is located at:

  **app/src/main/res/raw/default_heroes.json**

Do not search elsewhere. Load this file directly.

If the file does not exist at this exact path, fall back to visual
recognition only, but note the missing file prominently in the report.

Parse the JSON; it should contain an object where keys are hero names (e.g.,
"ruby") and values are either:
  - A string URL pointing to the CDN portrait image, or
  - An object with a "url" field (and optionally other metadata).

Example structure:
  {
    "ruby": "https://cdn.mlbb.com/heroes/ruby.png",
    "chou": "https://cdn.mlbb.com/heroes/chou.png",
    ...
  }

You may need to download the official portrait images from these URLs to use
for comparison. If you have network access, fetch them. If not, and the
images are not available locally, then use the JSON only for hero name
mapping after visual identification.

-------------------------------------------------------------------
STEP 1 — UNDERSTAND THE UI LAYOUT
-------------------------------------------------------------------

The MLBB draft phase screen has two mirrored columns of player slots:

LEFT SIDE (your team, 5 slots stacked vertically):
  - Located on the far-left edge of the screen
  - Each row left-to-right: [hero portrait] [flag icon] [rank icon] | [username]
  - The hero portrait is the LEFTMOST image element per row

RIGHT SIDE (enemy team, 5 slots, labeled "Player 1"–"Player 5"):
  - Mirrored on the far-right edge
  - Each row: [label] | [rank icon] [flag icon] [hero portrait]
  - The hero portrait is the RIGHTMOST image element per row

IGNORE the center area (hero selection grid, ban strip, lane tabs, match stats).

-------------------------------------------------------------------
STEP 2 — LOCATE EVERY HERO PORTRAIT
-------------------------------------------------------------------

For each input screenshot, find all filled player slots (up to 10 per image).
Use these proportional regions relative to full image dimensions (W x H):

  Region           X range          Y centers (rows 1 to 5)
  ---------------------------------------------------------------
  Left portraits   0.01W – 0.11W    0.16H, 0.30H, 0.44H, 0.58H, 0.72H
  Right portraits  0.89W – 0.99W    0.16H, 0.30H, 0.44H, 0.58H, 0.72H
  Portrait height  ~0.11H per row

Adjust boundaries based on actual visible edges — these are estimates.
Add a 3–5 px buffer on all sides of each detected portrait region.

For each detected portrait, also attempt to identify the hero's name from:
  - Any visible text label near the slot
  - Visual appearance using your MLBB hero knowledge
  - (NEW) By comparing with the official portraits from the JSON file

-------------------------------------------------------------------
STEP 3 — CROP EACH PORTRAIT
-------------------------------------------------------------------

For each detected, filled slot:
  - Crop the rectangular region containing only the hero portrait
  - Exclude country flag, rank icon, and username text
  - Output format: PNG, square or near-square crop (aspect ratio ~1:1)

-------------------------------------------------------------------
STEP 3.5 — COMPARE WITH OFFICIAL PORTRAITS (NEW)
-------------------------------------------------------------------

For each cropped candidate:

  1. Load the official portrait images from the JSON file at
     `app/src/main/res/raw/default_heroes.json`. For each entry, either
     download the image from the URL or use a locally cached copy.

  2. Compare the candidate crop against each official portrait using a
     visual similarity metric (e.g., perceptual hash, feature matching, or
     simple pixel‑wise difference if sizes are normalized).

  3. If the best matching official portrait has a similarity score above a
     threshold (e.g., 0.8), then:
       - Accept the crop as valid.
       - Assign the hero name from that official entry.
       - Optionally, you may refine the crop to match the official image's
         aspect ratio or bounding box if the candidate is slightly off.

  4. If no match is found (or if the JSON file is missing), fall back to
     visual identification and/or text‑based name detection as originally
     described.

  5. If the matching confidence is below 0.5, skip that candidate (report as
     "low confidence").

  6. If the candidate matches an official portrait but the hero name from
     the JSON differs from any previously detected text, trust the official
     match (it is the ground truth).

-------------------------------------------------------------------
STEP 4 — DEDUPLICATE
-------------------------------------------------------------------

Before saving, deduplicate across ALL crops (including across multiple
screenshots if more than one image is provided):

Deduplication rules:
  1. PRIMARY: Match by hero name (using the official name from the JSON if
     available).
     If two crops share the same identified hero name, keep only the FIRST
     occurrence; discard all subsequent duplicates.
  2. SECONDARY (if name is uncertain): Compare visually.
     If two portraits show the same character art (e.g., same official
     match), treat them as duplicates and keep only one.
  3. A hero appearing on both the left side AND right side still counts as
     a duplicate — keep one copy only.

After deduplication you should have at most one image per unique hero.

-------------------------------------------------------------------
STEP 5 — NAME EACH FILE
-------------------------------------------------------------------

Use this naming convention for saved PNG files:

  <hero_name>.png

Rules:
  - Lowercase
  - Spaces replaced with underscores
  - Remove special characters
  - Use the official hero name from the JSON if a match was found; otherwise
    use the visually identified name.
  - Examples:
      ruby.png
      chou.png
      layla.png
      unknown_slot_left_2.png   (fallback when hero cannot be identified)

If the official JSON provides multiple names or aliases, use the primary key.

-------------------------------------------------------------------
STEP 6 — CREATE THE TAR.GZ ARCHIVE
-------------------------------------------------------------------

Bundle all deduplicated PNG files into a single compressed archive.

Archive name:
  mlbb_hero_crops_<YYYYMMDD>.tar.gz
  (use today's date or a timestamp)

Archive internal structure (flat folder, no sub-directories):
  mlbb_hero_crops_20260701/
    ruby.png
    chou.png
    layla.png
    ...

Shell commands (if running in a shell environment):
  mkdir -p mlbb_hero_crops_<date>
  # move/copy all PNG files into that folder
  tar -czf mlbb_hero_crops_<date>.tar.gz mlbb_hero_crops_<date>/

-------------------------------------------------------------------

-------------------------------------------------------------------

After completing all steps, output a plain-text summary:

  === MLBB Hero Crop Report ===
  Screenshots processed : <N>
  Total slots detected  : <N>
  Duplicates removed    : <N>
  Unique heroes saved   : <N>
  JSON file used        : app/src/main/res/raw/default_heroes.json (or "Not found")

  Fallback (no match)   : <N> crops identified visually

    - ruby    -> ruby.png (matched with CDN)
    - chou    -> chou.png (matched with CDN)
    - layla   -> layla.png (matched with CDN)
    - unknown_left_2 -> unknown_slot_left_2.png (no CDN match)

  Archive: mlbb_hero_crops_<date>.tar.gz (<size>)
  =============================

-------------------------------------------------------------------
EDGE CASES
-------------------------------------------------------------------

  - Empty slot (no hero selected yet): skip entirely, do not create a file.
  - Low-confidence crop (similarity < 0.5 to any official portrait, and no
    visual/text certainty): skip and note in the report.
  - Duplicate on same screenshot: keep only the first occurrence.
  - No portraits found: return {"error": "no_hero_portraits_detected"}
  - Not an MLBB draft phase screenshot: return {"error": "not_a_draft_phase_screenshot"}
  - JSON file not found at `app/src/main/res/raw/default_heroes.json`: proceed without official matching, but warn in report.

-------------------------------------------------------------------
ADDITIONAL NOTES FOR THE AI
-------------------------------------------------------------------

- The JSON path is **fixed**. Look for `app/src/main/res/raw/default_heroes.json`
  relative to the repository root. Do not search anywhere else.
- If the JSON contains relative paths (e.g., `"portraits/ruby.png"`), treat
  them as relative to the JSON file's directory (i.e., `app/src/main/res/raw/`).
- If you have network access, download the official portraits and cache them
  for the duration of the task. If not, and only URLs are available, use the
  URL as the identifier and rely on visual recognition for matching (but
  you can still use the hero names from the JSON for naming).
- For similarity, consider resizing both candidate and official portrait to
  a common size (e.g., 128x128) and computing a simple mean squared error or
  using a pretrained feature extractor if available. The exact method is at
  your discretion; the goal is to reliably match the correct hero.
